# Практическое задание 1 — Основы многопоточности в .NET

## Описание
В рамках практического задания реализована система последовательной и параллельной обработки большого массива числовых данных с использованием базового API потоков (`Thread`) 

## Структура проекта

Проект состоит из следующих файлов:

### 1. DataProcessor.cs
Класс для обработки данных.

**Реализованные методы:**

- `decimal[] ProcessDataSequential(decimal[] data)`  
  Последовательно обрабатывает массив данных.  
  Для каждого элемента выполняется вычисление:  
  `Math.Sqrt(x) * Math.Log10(x + 1)`  
  (с дополнительными повторными вычислениями для увеличения вычислительной нагрузки).

- `decimal[] ProcessDataParallel(decimal[] data, int threadCount)`  
  Параллельно обрабатывает массив данных.  
  Массив делится на части по количеству потоков.  
  Каждый поток обрабатывает свой диапазон элементов.  
  Используется `Thread` и `CountdownEvent` для синхронизации завершения потоков.  
  Предусмотрена обработка исключений внутри потоков.

---

### 2. PerformanceMeter.cs
Класс для измерения производительности и сравнения результатов.

**Реализованные методы:**

- `(long timeMs, decimal[] result) MeasureExecutionTime(Func<decimal[]> action, string name)`  
  Измеряет время выполнения переданного метода обработки данных с помощью `Stopwatch`.

- `bool CompareResults(decimal[] r1, decimal[] r2, decimal tolerance = 0.0001m)`  
  Сравнивает результаты последовательной и параллельной обработки с заданной точностью.

---

### 3. Program.cs
Точка входа приложения и тестирование производительности.

**Что реализовано:**

- генерация массива из 10 000 000 элементов в диапазоне от 1.0 до 1000.0  
- использование фиксированного seed для `Random` (42)  
- запуск последовательной обработки и измерение времени  
- запуск параллельной обработки на 4 потоках и измерение времени  
- сравнение результатов двух методов  
- вывод итоговой статистики в консоль  

---

## Выводы

В результате выполнения задания была реализована последовательная и параллельная обработка данных с использованием базового API потоков.  
Параллельная версия показала ускорение в 3.5 раза по сравнению с последовательной при использовании 4 потоков.  
Во время выполнения параллельной версии в диспетчере задач наблюдается загрузка всех ядер процессора.  
Полученные результаты демонстрируют преимущество параллельной обработки для вычислительно нагруженных задач при больших объёмах данных и подтверждают корректность реализации многопоточности в рамках условий задания
